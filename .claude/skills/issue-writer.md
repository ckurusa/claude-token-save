# issue-writer

`token_report.py`와 `token_report.html`의 현재 상태를 분석하고, 개선이 필요한 항목을 GitHub 이슈로 등록합니다.

## 트리거

- `/issue-writer` 입력 시
- "이슈 만들어줘", "약점 분석해서 이슈 등록해줘", "개선 이슈 생성" 같은 요청

## 실행 순서

### 1단계 — 현재 상태 파악

```bash
python token_report.py
```

리포트 결과와 함께 `token_report.py`, `token_report.html` 코드를 검토한다.

### 2단계 — 약점 목록 작성

다음 기준으로 개선 항목을 찾는다:
- 기능 누락 (예: 여러 세션 집계, 추세 그래프 없음)
- 사용성 불편 (예: 파일 경로 수동 입력 필요)
- 정확도 문제 (예: 모델 가격 업데이트 필요)
- 코드 품질 (예: 에러 처리 미흡)

### 3단계 — 이슈 등록

```bash
gh issue create \
  --title "<제목>" \
  --body "<본문: 배경/현상/체크리스트>" \
  --label "enhancement"
```

**이슈 본문 형식:**
```
## 배경
왜 이 개선이 필요한지 한 줄.

## 현상 / 문제
구체적 증상 또는 누락 기능.

## 작업 항목
- [ ] 항목 1
- [ ] 항목 2
```

### 4단계 — 결과 보고

등록된 이슈 URL을 사용자에게 전달한다.
