# issue-runner

GitHub에 등록된 이슈를 확인하고, 우선순위가 높은 것을 골라 실제 코드를 수정한 뒤 이슈를 닫습니다.

## 트리거

- `/issue-runner` 입력 시
- "이슈 처리해줘", "이슈 작업 시작", "열린 이슈 처리" 같은 요청

## 실행 순서

### 1단계 — 이슈 목록 확인

```bash
gh issue list --state open
```

열린 이슈 번호·제목·레이블을 사용자에게 보여준다.

### 2단계 — 이슈 선택

사용자가 번호를 지정하지 않으면, 가장 오래된(번호 가장 작은) 이슈를 제안한다.

```bash
gh issue view <번호>
```

이슈 본문의 **작업 항목** 체크리스트를 확인한다.

### 3단계 — 구현

체크리스트 항목을 순서대로 처리한다.
- 코드 변경은 최소 단위로, 관련 파일만 수정
- 각 항목 완료 시 진행 상황 보고

### 4단계 — 변경 사항 커밋 & 푸시

```bash
git add <변경된_파일>
git commit -m "fix: <이슈 제목 요약> (closes #<번호>)"
git push
```

### 5단계 — 이슈 닫기

```bash
gh issue close <번호> --comment "구현 완료. 커밋: <커밋_해시>"
```

### 6단계 — 결과 보고

닫힌 이슈 URL과 변경된 파일 목록을 사용자에게 전달한다.
