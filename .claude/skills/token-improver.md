# token-improver

네 스킬(`token-report` → `issue-writer` → `issue-runner` → `doc-optimizer`)을 순서대로 실행해 프로젝트를 한 사이클 개선합니다.

## 트리거

- `/token-improver` 입력 시
- "프로젝트 개선해줘", "한 번에 다 돌려줘", "풀 사이클 실행" 같은 요청

## 실행 흐름

```
[1] 현황 파악 (token-report)
        ↓
[2] 약점 이슈 등록 (issue-writer)  ← 비용이 70% 이상일 때만
        ↓
[3] 이슈 하나 처리 (issue-runner)  ← 열린 이슈가 있을 때만
        ↓
[4] 문서 정리 (doc-optimizer)
        ↓
    완료 리포트
```

## 단계별 상세

### 1단계 — 현황 파악

```bash
python token_report.py
```

결과를 보고 비용 상태를 판단한다:
- 🟢 70% 미만: 2단계 건너뛰고 3단계로
- 🟡🔴 70% 이상: 2단계 실행

### 2단계 — 약점 이슈 등록 (조건부)

`issue-writer` 스킬을 실행한다.
- `token_report.py`와 `token_report.html` 코드를 검토해 개선 항목 도출
- `gh issue create`로 이슈 등록
- 이미 같은 주제의 열린 이슈가 있으면 건너뛴다

### 3단계 — 이슈 하나 처리 (조건부)

```bash
gh issue list --state open
```

열린 이슈가 있으면 `issue-runner` 스킬을 실행한다.
- 이슈 하나만 처리한다 (과도한 변경 방지)
- 처리 후 커밋·푸시·이슈 종료까지 완료

### 4단계 — 문서 정리

`doc-optimizer` 스킬을 실행한다.
- SSOT·Minimum·Freshness 기준으로 세 문서 점검
- 문제가 없으면 "문서 양호" 메시지만 출력하고 넘어간다

### 완료 리포트

```
✅ token-improver 완료
──────────────────────────
비용 상태   : 🟢 양호 ($0.82 / $1.00)
이슈 등록   : 건너뜀 (70% 미만)
이슈 처리   : #3 완료 → 닫음
문서 정리   : CLAUDE.md 1건 수정
──────────────────────────
```

## 주의사항

- 각 단계는 독립적이다. 한 단계가 실패해도 다음 단계를 계속 진행한다.
- 이슈 처리(3단계)는 한 번에 하나만 한다. 여러 이슈를 한꺼번에 처리하려면 `/issue-runner`를 직접 반복 실행한다.
- 전체 사이클이 끝난 후 사용자에게 요약을 전달한다.
