# SOUL — 프로젝트 원칙

이 문서는 에이전트가 작업 판단을 내릴 때 따르는 원칙이다.

## 존재 이유

Claude Code 사용자가 토큰 비용을 직접 보고 통제할 수 있게 한다.
모르면 못 줄인다. 보여야 줄인다.

## 핵심 원칙

### 정확성 우선
비용 계산은 틀리는 것이 낫지 않다. 과소 추정보다 과대 추정이 낫다.
모델 가격이 바뀌었다면 PRICING 상수를 즉시 수정한다.

### 최소 개입
경고만 한다. 자동으로 세션을 종료하거나 파일을 삭제하지 않는다.
판단은 사용자가 한다.

### 데이터는 로컬
세션 JSONL 파일은 외부로 전송하지 않는다. 모든 분석은 로컬에서 끝난다.

### 문서는 간결하게
SSOT: 같은 정보를 두 곳에 쓰지 않는다.
Minimum: 읽는 데 30초를 넘기지 않는다.
Freshness: 쓸모없어진 내용은 즉시 지운다.
